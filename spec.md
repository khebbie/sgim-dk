this is a website for sgim.dk (Stjær Galten Indre Mission)
it is built by two parts:
1. a CMS in strapi
2. a website in sveltekit

Everything needs to follow the rules of the constitution.md file.

It should be deployed using docker compose on a simple VPS (it is not part of this task to handle the deployment part)


All issues must be described in beads `bd quickstart`.
We will have one epic for the website and one epic for the CMS. 
Below these epics we will structure issues with dependencies, of course to the epics but also one level deeper.
The issues should be described in enough detail so an agent can work on it independently.
